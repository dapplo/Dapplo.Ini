# UI Configuration Framework

`Dapplo.Ini.Ui` is the UI configuration layer built on top of Dapplo.Ini.  You annotate
your existing `IIniSection` interfaces with declarative attributes, and the framework (plus
its source generator) transforms those annotations into compile-time UI descriptors that
any UI renderer (Avalonia, WPF, etc.) can consume to automatically build a settings window
with tabs, groups, controls, translations, conditional visibility/enable, and
apply-or-rollback semantics.

---

## Quick start

### 1 — Install the packages

```shell
dotnet add package Dapplo.Ini
dotnet add package Dapplo.Ini.Generator
dotnet add package Dapplo.Ini.Ui
dotnet add package Dapplo.Ini.Ui.Generator
```

`Dapplo.Ini.Ui.Generator` is the Roslyn source generator that produces a compile-time
`*UiDescriptor` class for every annotated section interface.

---

### 2 — Annotate your section interface

```csharp
using Dapplo.Ini;
using Dapplo.Ini.Ui.Attributes;
using Dapplo.Ini.Ui.Enums;

// ── INI section that also describes its settings-window page ─────────────────

[IniSection("Network")]
[UiPage(Title = "Network", Category = "Advanced", Order = 10)]
[UiChangeMode(UiChangeMode.OnConfirm)]   // all controls buffer changes until OK
public interface INetworkSettings : IIniSection, ITransactional
{
    [DefaultValue(false)]
    [UiLabelKey("network_use_proxy_label", DescriptionKey = "network_use_proxy_tip")]
    bool UseProxy { get; set; }

    [DefaultValue("")]
    [UiGroup("Proxy", Order = 5)]
    [UiConditionalVisibility(nameof(UseProxy))]   // hidden when UseProxy = false
    [UiConditionalEnable(nameof(UseProxy))]        // disabled when UseProxy = false
    [UiLabelKey("network_proxy_host_label")]
    string ProxyHost { get; set; }

    [DefaultValue(8080)]
    [UiControl(UiControlType.UpDown, Minimum = 1, Maximum = 65535)]
    [UiGroup("Proxy")]
    [UiConditionalVisibility(nameof(UseProxy))]
    [UiConditionalEnable(nameof(UseProxy))]
    int ProxyPort { get; set; }
}
```

Key attributes:

| Attribute | Scope | Purpose |
|-----------|-------|---------|
| `[UiPage]` | interface | Sets page title, category (tree node), order, optional icon |
| `[UiChangeMode]` | interface / property | `Immediate` (default) or `OnConfirm` (buffer until OK) |
| `[UiControl]` | property | Override inferred control; set Minimum / Maximum / Increment / Unit |
| `[UiGroup]` | property | Group several properties under a labelled box; supports group-level visibility/enable |
| `[UiLabelKey]` | interface / property | i18n key → label text; optional `DescriptionKey` for tooltip |
| `[UiConditionalVisibility]` | property | Show/hide based on a `bool` property; supports `Invert` |
| `[UiConditionalEnable]` | property | Enable/disable based on a `bool` property; supports `Invert` |
| `[UiOrder]` | property | Explicit ordering within the group/page |
| `[UiLayout]` | assembly / interface | Window-level: `HorizontalTabs`, `VerticalTabs`, or `SinglePage` |

### 3 — The source generator produces a compile-time descriptor

The `Dapplo.Ini.Ui.Generator` Roslyn source generator detects every `IIniSection`
interface that carries at least one UI attribute and emits a static descriptor class:

```csharp
// ── Auto-generated by Dapplo.Ini.Ui.Generator ────────────────────────────────

public static partial class NetworkSettingsUiDescriptor
{
    public static Dapplo.Ini.Ui.Metadata.UiPageMetadata Page { get; } =
        new Dapplo.Ini.Ui.Metadata.UiPageMetadata
        {
            SectionType = typeof(INetworkSettings),
            Title = "Network",
            Category = "Advanced",
            Order = 10,
            DefaultChangeMode = UiChangeMode.OnConfirm,
            Properties = new ReadOnlyCollection<UiPropertyMetadata>(new[]
            {
                new UiPropertyMetadata
                {
                    PropertyName = "UseProxy",
                    ControlType = UiControlType.CheckBox,
                    LabelKey = "network_use_proxy_label",
                    DescriptionKey = "network_use_proxy_tip",
                    ChangeMode = UiChangeMode.OnConfirm,
                },
                new UiPropertyMetadata
                {
                    PropertyName = "ProxyHost",
                    ControlType = UiControlType.TextBox,
                    GroupName = "Proxy",
                    Order = 5,
                    LabelKey = "network_proxy_host_label",
                    VisibilityConditionProperty = "UseProxy",
                    EnableConditionProperty = "UseProxy",
                    ChangeMode = UiChangeMode.OnConfirm,
                },
                // … ProxyPort …
            })
        };
}
```

You never need to write this by hand. Any UI renderer calls
`NetworkSettingsUiDescriptor.Page` to get the full description without any runtime
reflection.

### 4 — Wire translated labels via LanguageConfig

```csharp
var language = await LanguageConfigBuilder
    .ForBasename("MyApp")
    .AddSearchPath(AppContext.BaseDirectory)
    .RegisterSection<INetworkLanguage>(new NetworkLanguageImpl())
    .BuildAsync();

// Resolve the label text for a property:
string label = language.GetSection<INetworkLanguage>().GetValue("network_use_proxy_label");
```

The `LabelKey` values in the descriptor map 1-to-1 to keys in your `.ini` language files:

```ini
; MyApp.en-US.ini
[NetworkLanguage]
network_use_proxy_label = Use proxy server
network_use_proxy_tip   = Route all outgoing connections through the configured proxy
network_proxy_host_label = Proxy address
```

### 5 — Manage the settings session (apply / rollback)

```csharp
// At application startup or DI registration:
var manager = new UiSettingsManager();
manager.RegisterSection(networkSection);   // the IIniSection / ITransactional instance

// When the settings window opens:
manager.BeginSession();   // records rollback point via ITransactional.Begin()

// User changes controls … sections are modified in real time (Immediate)
// or buffered (OnConfirm)

// On OK / Apply:
manager.Apply();          // ITransactional.Commit() on all sections

// On Cancel / window close:
manager.Rollback();       // or EndSession() auto-rolls back if HasPendingChanges

// HasPendingChanges can be data-bound to enable/disable the Apply button:
bool dirty = manager.HasPendingChanges;
```

### 6 — React to control interactions

Implement `IUiControlEvents` in a `partial class` next to the generated `*Impl` class:

```csharp
// INetworkSettings partial – reacts to changes while the settings window is open
public partial class NetworkSettingsImpl : IUiControlEvents
{
    public void OnPropertyChanged(string propertyName, object? oldValue, object? newValue)
    {
        if (propertyName == nameof(INetworkSettings.UseProxy))
        {
            // e.g. automatically populate ProxyHost from an environment variable
            // the first time the user enables the proxy option.
            if (newValue is true && string.IsNullOrEmpty(ProxyHost))
                ProxyHost = Environment.GetEnvironmentVariable("HTTP_PROXY") ?? "";
        }
    }

    public void OnControlFocused(string propertyName)
    {
        if (propertyName == nameof(INetworkSettings.ProxyHost))
        {
            // Show inline help text, run a background connectivity check, etc.
        }
    }
}
```

All methods have default no-op implementations — override only what you need.

### 7 — Disable / hide constant-valued properties automatically

Properties loaded from a *constants* file (`IsConstant(key) == true`) are always rendered
as disabled in the settings UI, regardless of any `[UiConditionalEnable]` attribute.
This satisfies the issue requirement that constant values produce read-only (or hidden)
controls.

## Control types and inference

When `[UiControl]` is absent the generator infers the control type from the .NET property
type:

| Property type | Inferred control |
|---|---|
| `bool` / `bool?` | `CheckBox` |
| `enum` | `DropDown` |
| `int`, `long`, `double`, … | `UpDown` |
| `string` | `TextBox` |

Use `[UiControl]` to override — e.g. to render an `int` as a `Slider`:

```csharp
[UiControl(UiControlType.Slider, Minimum = 0, Maximum = 100, Unit = "%")]
int Volume { get; set; }
```

## Assembly-level layout

```csharp
// AssemblyInfo.cs (or any file in the consuming project)
[assembly: UiLayout(UiTabLayout.VerticalTabs, WindowTitle = "Application Settings")]
```

Supported layouts:

| `UiTabLayout` value | Effect |
|---|---|
| `HorizontalTabs` | Classic top-aligned tab strip (default) |
| `VerticalTabs` | Left-aligned tab strip or tree navigation |
| `SinglePage` | All sections stacked vertically on one scrollable page |

---

## Runtime metadata reader (optional)

If you need to discover page metadata dynamically at runtime (e.g. for a plugin host that
loads section interfaces at runtime) you can use `UiMetadataReader` instead of the
generated descriptor:

```csharp
UiPageMetadata meta = UiMetadataReader.ReadPage<INetworkSettings>();
// or
UiPageMetadata meta = UiMetadataReader.ReadPage(typeof(INetworkSettings));
```

This uses reflection and is slightly slower, but is otherwise equivalent to the
compile-time descriptor.
